Learn the basics of TypeScript and how it enhances JavaScript development.

Introduction to TypeScript

TypeScript is a strongly typed programming language that builds on JavaScript, giving you better tooling at any scale. It adds static type definitions to JavaScript, helping catch errors early and making your code more robust.

Why Use TypeScript?

TypeScript offers several advantages over plain JavaScript:

  1. Static Type Checking: Catch type-related errors at compile time rather than runtime
  2. Enhanced IDE Support: Better autocompletion, navigation, and refactoring tools
  3. Improved Readability: Types serve as documentation
  4. Safer Refactoring: The compiler catches errors when you change your code
  5. Modern JavaScript Features: Use the latest ECMAScript features with backward compatibility

Getting Started

To start using TypeScript, you'll need to install it first:

1# Using npm
2npm install -g typescript
3

4# Using yarn
5yarn global add typescript

Create a simple TypeScript file (e.g., 

1hello.ts
):

1function greet(name: string): string {
2  return `Hello, ${name}!`;
3}
4

5console.log(greet("TypeScript"));

Compile it to JavaScript:

1tsc hello.ts

This will generate a 

1hello.js
file that you can run with Node.js.

1// Boolean
2let isDone: boolean = false;
3

13// Array

17// Tuple

Interfaces

1interface User {
2  id: number;
3  name: string;
4  email: string;
5  age?: number; // Optional property
6  readonly createdAt: Date; // Read-only property
7}
8

9function createUser(user: User): User {
10  return user;
11}
12

13const newUser = createUser({
14  id: 1,
15  name: "John Doe",
16  email: "john@example.com",
17  createdAt: new Date()

Classes

TypeScript supports class-based object-oriented programming:

1class Person {
2  // Properties
3  private name: string;
4  protected age: number;
5  
6  // Constructor
7  constructor(name: string, age: number) {
8    this.name = name;
9    this.age = age;
10  }
11  
12  // Methods
13  public greet(): string {
14    return `Hello, my name is ${this.name} and I am ${this.age} years old.`;
15  }
16}
17

18// Inheritance
19class Employee extends Person {
20  private department: string;
21  
22  constructor(name: string, age: number, department: string) {
23    super(name, age);
24    this.department = department;
25  }
26  
27  public getDetails(): string {
28    return `${this.greet()} I work in the ${this.department} department.`;
29  }
30}
31

32const employee = new Employee("Alice", 30, "Engineering");
33console.log(employee.getDetails());

Generics

Generics allow you to create reusable components:

1function identity<T>(arg: T): T {
2  return arg;
3}
4

5let output1 = identity<string>("myString");
6let output2 = identity<number>(100);
7

8// Generic interface
9interface GenericResponse<T> {
10  data: T;
11  status: number;
12  message: string;
13}
14

15// Using the generic interface
16const userResponse: GenericResponse<User> = {
17  data: newUser,
18  status: 200,
19  message: "Success"
20};

Type Assertions

Type assertions are a way to tell the compiler "trust me, I know what I'm doing":

1// Using angle-bracket syntax
2let someValue: any = "this is a string";
3let strLength: number = (<string>someValue).length;
4

5// Using as syntax (preferred in JSX)
6let otherValue: any = "another string";
7let otherLength: number = (otherValue as string).length;

Advanced Types

4    return value.trim();

Conclusion

TypeScript enhances JavaScript development by adding static types and modern language features. It helps catch errors early, improves code quality, and provides better tooling support. As your projects grow in size and complexity, TypeScript becomes increasingly valuable for maintaining code quality and developer productivity.
